Hash Table

Get Started

Definition

Hash table is the data structure that used to store entries (key-value pairs) for storing and retrieving information quickly.

Hash table offers a combination of efficient lookup, insertion, deletion operations O(1).

It utilizes a hash function to compute an index into an array of buckets, where entries can be found or inserted.

Use hash function that converts a non-integer key into a non-negative integer key.

Hash Function

The hash function is used to transform the key into the index

For numeric key

Divide the key by the size of available array, array_size, and take the remainer.

For alphanumeric key

Divide sum of (ASCII codes) of characters by size of available array, and take the remainer.

Choosing a hash function

Quick computation

Computing hash value should be quick (constant time).

Handling Hash collision

Less hash collision which is a situation that occurs when two keys have identical hash values.

Uniform distribution

A good hash function should distribute the keys uniformly into the slots in the table.

Perfect hash function

A hash function that map each key into a unique bucket is referred as a perfect hash function.

Hash value

A hash function returns positive integer value which is called a hash code.

How does hash table work


• In hashing, keys are converted into fixed-size integers by using hash function. The idea of hasing is to distribute entries (key/value pairs) uniformly across an array of bucket.

• Hashing is implemented in two steps:
+A key is converted into an integer by using a hash function.
+The key is stored in the hash table where it can be quickly retrieved using hashed key. index = key % array_size;

• Hence, the key is reduced to an index with range from 0 to array_size – 1 by using the modulo operator(%) .By using that key, you are able to quickly access the element in O(1) time.

Separate chaining

• Separate chaining is a method which is utilized to handle hash collision.

•Technically, Hash table in separate chaining is usually implemented with the aid of linked list.

• Each bucket of the hash table contains a collection of entries.

• Each bucket points to a linked list of entries.

• To look up an entry, you find its bucket and then walk the list until you find an entry with the matching key.

• For separate chaining, the worst case scenario is when all the entries are inserted into the same linked list => the lookup procedure may have to scan all its entries and time complexity would be O(n).

Open addressing

※ In open adressing, all the data go inside the table.
※ Size of the table must be greater than or equal to the total number of keys
※ There are three common collision resolution strategies

Linear Probing

We scan forwards one index at a time for the next empty/deleted slot (wrapping around when we have reached the last slot) whenever there is a collision.

Quadratic Probing

• We look for i2th slot in i'th iteration if the given hash vale x collides in the hash table.

• Let hash(x) be the slot index computed using the hash function.
+ If the slot hash(x) % S is full, then we try (hash(x) + 1*1) % S.
+ If (hash(x) + 1*1) % S is also full, then we try (hash(x) + 2*2) % S.
+ If (hash(x) + 2*2) % S is also full, then we try (hash(x) + 3*3) % S.
+ This process is repeated for all the values of i until an empty slot is found.

Double Hashing

• Double hashing uses the idea of applying a second hash function to key when a collision occurs
• First hash function is typically hash1(key) = key % TABLE_SIZE
• A popular second hash function is : hash2(key) = PRIME – (key % PRIME) where PRIME is an arbitrary prime number which is smaller than theTABLE_SIZE.
• There are a couple of requirements for the second function:
+ It must never evaluate to 0
+ Must make sure that all cells can be probed

Applications

※ Associative arrays: Hash tables are commonly used to implement many types of in-memory tables. They are used to implement associative arrays (arrays whose indices are arbitrary strings or other complicated objects).

※ Database indexing: Hash tables may also be used as disk-based data structures and database indices (such as in dbm).

※ Caches: Hash tables can be used to implement caches i.e. auxiliary data tables that are used to speed up the access to data, which is primarily stored in slower media.

※ Object representation: Several dynamic languages, such as Perl, Python, JavaScript, and Ruby use hash tables to implement objects.

※ Hash Functions are used in various algorithms to make their computing faster
